# S1.6 — Retrieval Layer · Sequence Diagrams

All diagrams use Mermaid and reflect the implementation in `backend/retrieval/`.

---

## 1. Standard semantic retrieval (happy path)

```mermaid
sequenceDiagram
    participant C as Caller (RAG / API)
    participant R as Retriever
    participant X as QueryExpander
    participant I as VectorIndex (S1.5)
    participant F as filters
    participant K as Reranker
    participant M as MetricsSink

    C->>R: retrieve(RetrievalQuery)
    R->>M: incr(queries)
    opt enable_query_expansion
        R->>X: safe_expand(query)
        X-->>R: ExpansionResult(query', terms)
    end
    R->>I: search(query_text, tenant_id, top_k)
    I-->>R: [IndexCandidate] (similarity-ordered)
    R->>M: incr(candidates, n)
    R->>R: apply min_score floor
    R->>F: apply_all(tenant_id, principals, clause)
    Note over F: tenant isolation (raise) → ACL drop → metadata drop
    F-->>R: (filtered hits, FilterStats)
    R->>M: incr(acl_dropped / filter_dropped)
    opt enable_rerank
        R->>K: safe_rerank(query', filtered)
        K-->>R: (reranked hits, degraded?)
    end
    R->>R: paginate(offset, limit)
    R->>M: timing(latency) + incr(success) + record_event
    R-->>C: RetrievalResult(hits, PageInfo, diagnostics)
```

## 2. Tenant isolation breach (fail-closed)

```mermaid
sequenceDiagram
    participant C as Caller
    participant R as Retriever
    participant I as VectorIndex (S1.5)
    participant F as filters
    participant M as MetricsSink

    C->>R: retrieve(query tenant=t1)
    R->>I: search(tenant_id=t1, top_k)
    I-->>R: [cand(t1), cand(t2)]  %% upstream scoping bug
    R->>F: enforce_tenant_isolation(hits, t1)
    F--xR: raise TenantIsolationError(found_tenant=t2)
    R->>M: incr(failure) + record_event(success=false)
    R--xC: TenantIsolationError
```

The breach is never swallowed: a foreign-tenant record signals an upstream bug
and must surface loudly.

## 4. Non-fatal degradation (reranker fails → fall back to vector order)

```mermaid
sequenceDiagram
    participant C as Caller
    participant R as Retriever
    participant K as Reranker (LLM, future)
    participant M as MetricsSink

    C->>R: retrieve(query, enable_rerank=true)
    Note over R: vector search + filtering already succeeded
    R->>K: safe_rerank(query, filtered)
    K--xR: exception
    R->>R: keep vector-similarity order, degraded=true
    R->>M: incr(degraded)
    R-->>C: RetrievalResult(diagnostics.degraded=true, reranked=false)
```

Query expansion follows the identical degrade-don't-fail pattern via
`safe_expand`.

## 5. Hybrid retrieval with score fusion

```mermaid
sequenceDiagram
    participant C as Caller
    participant H as HybridRetriever
    participant R as Retriever (base)
    participant FI as _FusingIndex
    participant V as VectorIndex (S1.5)
    participant KW as KeywordIndex

    C->>H: retrieve(query with filters)
    H->>H: inflate top_k (overfetch_factor)
    H->>R: retrieve(query')
    R->>FI: search(query_text, tenant_id, top_k)
    FI->>V: search(...)
    V-->>FI: [IndexCandidate] (vector ranks)
    FI->>KW: search_ranked(...)
    KW-->>FI: [chunk_id] (keyword ranks)
    FI->>FI: RRF fuse(vector_rank, keyword_rank)
    FI-->>R: [IndexCandidate] (fused scores)
    Note over R: identical tenant/ACL/filter/rerank/paginate pipeline
    R-->>H: RetrievalResult
    H-->>C: RetrievalResult
```

Because `_FusingIndex` satisfies the `VectorIndex` contract, hybrid retrieval
reuses every downstream guarantee of the base retriever.
