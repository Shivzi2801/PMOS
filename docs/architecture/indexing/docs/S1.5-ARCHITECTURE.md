# S1.5 — Index Fan-Out — Architecture

**Wave 1 / Slice 1.5**
**Status:** design + reference implementation
**Depends on:** S1.1 (Connector SDK, secrets/KMS), S1.2 (canonical normalization, PII screening, quarantine), S1.3 (extraction, typed atoms), S1.4 (canonical entities, entity resolution, provenance, **source ACL enforcement**)

---

## 1. Purpose and scope

S1.5 turns a *canonical document* (the post-normalization, post-resolution artifact produced by S1.2–S1.4) into *indexed, retrievable units* — **document chunks** — and fans those chunks out to one or more vector indexes. It also provides the safety net (a nightly reconciler) that keeps the indexes consistent with the canonical source over time.

This slice owns four responsibilities:

1. The `document_chunks` contract — the atomic unit of indexing.
2. The chunking pipeline — sizing, overlap, and metadata enrichment.
3. Content-hash deduplication — never index the same content twice within a tenant.
4. The fan-out orchestrator — chunk → dedup → embed → partition → dispatch — plus a vector-store abstraction (with a Qdrant *contract*, not a client), a payload partitioning strategy, a nightly reconciler, metrics, and failure handling.

Out of scope (explicitly): the embedding model itself (injected as a callback), a real Qdrant client, retrieval/query ranking, and any production infrastructure wiring. These belong to other slices or to deployment.

---

## 2. Position in the PMOS pipeline

```
S1.1 Connectors ─▶ S1.2 Normalize/Screen ─▶ S1.3 Extract ─▶ S1.4 Resolve/ACL ─▶ [S1.5 Index Fan-Out] ─▶ (retrieval, future)
```

S1.5 consumes a `CanonicalDocumentView` — a deliberately narrow projection of the canonical document. It depends only on: `tenant_id`, `document_id`, `content`, `entity_ids` (from S1.4 resolution), `source_acl` (from S1.4 ACL enforcement), and a little provenance metadata (`source_type`, `normalized_at`). It does **not** import the full S1.2/S1.4 document types; the narrow view keeps indexing decoupled and independently testable.

## 3. The `document_chunks` contract

A `DocumentChunk` is immutable (frozen dataclass) and self-validating. Required fields, exactly as specified by the slice:

| Field | Type | Meaning |
|---|---|---|
| `chunk_id` | `str` | Deterministic UUIDv5 over `(tenant_id, document_id, ordinal)` |
| `tenant_id` | `str` | Tenant scope — the root partition key. Never empty. |
| `document_id` | `str` | Source canonical document |
| `entity_ids` | `tuple[str, ...]` | Resolved canonical entities (S1.4) present in the chunk |
| `source_acl` | `frozenset[str]` | Principals allowed to retrieve this chunk (copied from S1.4) |
| `content` | `str` | Normalized, PII-screened text (S1.2) |
| `content_hash` | `str` | Lowercase hex SHA256 of normalized content — the dedup/identity key |
| `metadata` | `Mapping` | Enrichment (ordinal, spans, source_type, …) |
| `created_at` | `datetime` | Timezone-aware UTC |

**Key invariants** (enforced in `__post_init__`, raising `ContractViolation`):

- `tenant_id` is mandatory and non-empty. There is no global/un-tenanted chunk.
- `content_hash` must be a 64-char lowercase hex SHA256. The contract checks shape only; `hashing.py` is the canonical producer.
- `source_acl` is a frozenset of principals. **An empty ACL means not-retrievable (fail-closed)**, never "public". `is_retrievable` reflects this.
- `created_at` must be timezone-aware.

**Content-safety design.** `content` carries text that may have contained PII before S1.2 screening. The contract exposes `safe_descriptor()` — an ACL-safe projection (chunk_id, tenant_id, document_id, content_hash, acl_size, created_at) that deliberately **omits `content` and `entity_ids`** — for any telemetry/log/report path, which has a wider blast radius than the source document. Nothing in S1.5 ever logs raw content.

## 4. Chunking pipeline

Three cooperating pieces:

**4.1 Sizing strategy (`ChunkSizing`)** — `target_size` (preferred), `max_size` (hard ceiling never exceeded), `min_size` (a trailing sliver below this is merged into its predecessor). Sizes are in characters for this slice; token-aware measurement is a pluggable future concern.

**4.2 Overlap strategy (`ChunkOverlap`)** — consecutive chunks share `overlap` trailing/leading characters so a fact straddling a boundary survives in at least one chunk. Overlap must be `< target_size`, which **guarantees forward progress** (the planner cannot loop).

**4.3 Boundary preference.** Rather than cut mid-word, `ChunkPlanner` searches backward from the hard end within a lookback window for the latest *soft* boundary — paragraph (`\n\n`), then line, sentence, clause, finally whitespace — falling back to a hard cut bounded by `max_size`. This produces semantically coherent chunks without an external tokenizer (no-dependency constraint).

**4.4 Identity + enrichment (`Chunker`).** `chunk_id` is a deterministic UUIDv5 of `(tenant_id, document_id, ordinal)`. Determinism is load-bearing: re-chunking the same document yields the same ids, which is how the reconciler distinguishes *missing* from *new* without a random-id side table. Metadata enrichment is additive and ACL-safe: `ordinal`, `span_start`, `span_end`, `char_len`, `overlap_prev`, plus carried `source_type` / `normalized_at`. Empty/whitespace documents raise `EmptyDocumentError`.

## 5. Content-hash deduplication

- **Hash** (`hashing.py`): SHA256 over a *normalized* byte form — Unicode NFC, per-line trailing-whitespace trim, `\n` line endings, UTF-8. Normalization means visually-identical content hashes identically. The hash covers **content only** — never metadata or ACL.
- **Scope:** dedup state is keyed by `(tenant_id, content_hash)`. **Dedup never collapses across tenants** — doing so would leak the existence of content across an ACL boundary. Identical text in two tenants is two distinct, separately-indexed chunks.
- **Suppression:** `Deduplicator.deduplicate` partitions a batch into `unique` vs `duplicates`, deduping both against prior runs (the `DedupStore`) and within the current batch (a document repeating a paragraph). Only `unique` chunks proceed to embedding/indexing.
- **Collision guard:** the store records the content *length* per hash; if a later chunk presents the same hash with a different length, `HashCollisionError` is raised. A true SHA256 collision is infeasible, so this is a defensive signal of a hashing bug or tampering — never retryable, always surfaced.
- **Metrics:** `chunks_deduplicated` counts suppressed chunks per tenant.

## 6. Index fan-out orchestrator

`IndexFanOut.index_document` runs the pipeline:

```
chunk ─▶ deduplicate ─▶ embed (injected) ─▶ build ACL-safe payload ─▶ dispatch to N targets
```

- **Fan-out = one chunk set, many targets.** The same unique set is dispatched to every registered `VectorStore` (e.g. a primary store plus a shadow store during migration).
- **Per-target isolation.** A failure on target B never rolls back a success on target A. Because upserts are keyed by `chunk_id` (idempotent), a later retry of B cannot duplicate in A. When some targets fail, a `PartialIndexError` carries per-target outcomes so a retry worker can retry **only** the failed targets.
- **Retry.** Transient (`retryable=True`) vector-store errors retry in-process with bounded exponential backoff (`RetryPolicy`). Non-retryable rejections short-circuit.
- **Embeddings are injected** as `Embedder = Callable[[Sequence[str]], Sequence[Sequence[float]]]`, so S1.5 carries no model dependency.
- A `reindex_chunks` repair entry point (used by the reconciler) writes a known-good chunk set directly, **bypassing dedup** — see §9.

## 7. Vector store abstraction & Qdrant contract

**`VectorStore` interface** (`vector_store.py`): `upsert`, `delete`, `fetch_ids` (for reconciliation), `search` (ACL-filtered), `health`. The interface is ACL-aware by construction — every read takes a `tenant_id`, and `search` additionally takes the caller's principals — so no concrete store can "forget" tenant scoping. An `InMemoryVectorStore` reference implementation enforces the same tenant/ACL invariants for tests.

**Qdrant contract** (`qdrant_contract.py`): a specification stub. Every method raises `NotImplementedError` — wiring a real `qdrant-client` is explicitly future infra work. It pins down the collection name, the required payload indexes, and the canonical payload/filters a real adapter must use.

---

## 8. Payload partitioning strategy

Single physical Qdrant collection, **logically partitioned by `tenant_id`**:

- Every payload carries `tenant_id`; a real adapter **must** index it and **must** AND a `tenant_id == caller` filter into every query. This is the primary isolation boundary.
- `acl` holds the source principals (sorted list). Queries **must** require overlap between the caller's principals and `acl`. **Empty `acl` ⇒ unreachable (fail-closed)** — never treated as public.
- Point `id == chunk_id` (deterministic) ⇒ idempotent upserts, directly addressable by the reconciler.

**ACL-safe payload structure.** `build_payload` is the *single sanctioned path* from a chunk to a stored payload, and it deliberately **excludes `content`**. The vector store holds only what is needed to search and to enforce access — `document_id`, `tenant_id`, `acl`, `entity_ids`, `content_hash`, `ordinal`, `source_type`, `created_at`. Original text is re-fetched from the canonical store at read time under the same ACL, so the wider-blast-radius vector store never becomes a second copy of sensitive text.

## 9. Nightly reconciler

The fan-out path can drop writes (partial failures, crashes, outages). The reconciler periodically compares the **expected** chunk set for each document against what each target actually holds, and repairs drift.

- **Authority:** an `ExpectedChunkSource` derives expected chunks by re-chunking the canonical document — pure and deterministic thanks to deterministic chunk_ids — and answers `expected_hash(point_id)` for stale detection.
- **Detections:** **missing** (expected id absent from a target), **stale** (present id whose authoritative hash differs from the freshly-computed expected hash — i.e. content changed at that ordinal), and **orphan** (present id no longer expected).
- **Repair:** orphans are deleted; missing/stale are repaired via `IndexFanOut.reindex_chunks`, which writes the authoritative expected chunks **directly, bypassing dedup**. This bypass is essential: the dedup ledger still lists the content as "seen," so the normal index path would write nothing even though the store genuinely lost the vector. Idempotent upserts make redundant writes harmless.
- **Report & resilience:** every finding lands in a `ReconciliationReport` with aggregate counters. One failing document does **not** abort the run — failures are counted (`reconciliation_failures`) and recorded, not thrown.

## 10. Metrics

Dependency-free `MetricsSink` (counter/histogram), labeled by `tenant_id` at minimum; labels never contain content or ACL principals.

| Metric | Type | Emitted by |
|---|---|---|
| `chunks_created` | counter | fan-out, after chunking |
| `chunks_deduplicated` | counter | fan-out, after dedup |
| `chunks_indexed` | counter | fan-out, per successful target |
| `reconciliation_failures` | counter | reconciler |
| `indexing_latency_ms` | histogram | fan-out (`LatencyTimer`) |

## 11. Failure handling (summary)

Full detail in **S1.5-FAILURE-HANDLING.md**. Errors share an `IndexingError` base carrying `retryable` and a `category`, plus ACL-safe `context()`. Headlines: partial index failures are isolated and reported per target; duplicate **hash collisions** raise the non-retryable `HashCollisionError`; reconciliation failures are counted and reported, never fatal to a run; retries use bounded exponential backoff and only apply to errors flagged `retryable`.

## 13. Constraints honored

Pure Python, standard library only; no external dependencies, no framework code, no production infrastructure code. The Qdrant client is intentionally absent (contract only). Embedding models are injected, not implemented.
