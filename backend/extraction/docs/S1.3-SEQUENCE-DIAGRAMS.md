# S1.3 — F-10 Extraction Engine — Sequence Diagrams

All diagrams use Mermaid. Render in any Mermaid-aware viewer.

---

## 1. Happy path — rule-based match

```mermaid
sequenceDiagram
    participant C as Caller (Ingestion)
    participant P as ExtractionPipeline
    participant V as Validator
    participant R as RuleBasedExtractor
    participant S as ConfidenceScorer
    participant K as AtomRanker
    participant M as MetricsSink

    C->>P: run(CanonicalDocument, context?)
    P->>V: validate_canonical_document(doc)
    V-->>P: ok
    P->>M: increment(documents_total)
    P->>R: extract(text, ctx)
    R-->>P: [FactAtom(conf>=0.90)]
    Note over P: Stage 1 produced atoms → skip Stage 2 & 3
    P->>S: apply(atom, RULE_BASED)
    S-->>P: validated atom
    P->>K: rank(atoms)
    K-->>P: deduped atoms
    P->>M: observe(latency_ms)
    P->>M: increment(atoms_total)
    P-->>C: ExtractionResult
```

## 2. Fallback path — heuristic stage

```mermaid
sequenceDiagram
    participant C as Caller
    participant P as ExtractionPipeline
    participant R as RuleBasedExtractor
    participant H as HeuristicExtractor
    participant S as ConfidenceScorer
    participant K as AtomRanker

    C->>P: run(doc)
    P->>R: extract(text, ctx)
    R-->>P: [] (no template matched)
    Note over P: Stage 1 empty → run Stage 2
    P->>H: extract(text, ctx)
    H-->>P: [FactAtom(0.60..0.80)]
    Note over P: Stage 3 (LLM) interface-only → not invoked
    P->>S: apply(atom, HEURISTIC)
    S-->>P: validated atom
    P->>K: rank(atoms)
    K-->>P: deduped atoms
    P-->>C: ExtractionResult
```

## 3. Malformed document (terminal failure)

```mermaid
sequenceDiagram
    participant C as Caller
    participant P as ExtractionPipeline
    participant V as Validator
    participant M as MetricsSink

    C->>P: run(None or invalid doc)
    P->>V: validate_canonical_document(doc)
    V-->>P: raise MalformedDocumentError
    P->>M: increment(failures_total, stage=validate)
    P-->>C: raise MalformedDocumentError
    Note over C: Caller routes document to dead-letter path
```

## 4. Isolated extractor fault (non-terminal)

```mermaid
sequenceDiagram
    participant P as ExtractionPipeline
    participant R as RuleBasedExtractor
    participant H as HeuristicExtractor
    participant M as MetricsSink

    P->>R: extract(text, ctx)
    R-->>P: raise RuntimeError
    P->>M: increment(failures_total, stage=rule_based)
    Note over P: Fault isolated; stage treated as empty
    P->>H: extract(text, ctx)
    H-->>P: [FactAtom...]
    Note over P: Document still produces a valid result
```

## 5. Cascade decision flow

```mermaid
flowchart TD
    A["CanonicalDocument"] --> B{"valid?"}
    B -->|"no"| F["MalformedDocumentError + failures_total"]
    B -->|"yes"| C["RuleBasedExtractor"]
    C --> D{"atoms found?"}
    D -->|"yes"| G["ConfidenceScorer"]
    D -->|"no"| E["HeuristicExtractor"]
    E --> G
    G --> H["AtomRanker (dedup + merge)"]
    H --> I["ExtractionResult"]
```
