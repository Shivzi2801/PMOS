# S1.3 ‚Äî F-10 Extraction Engine ‚Äî Architecture

**Wave 1 / Slice 1.3**
**Status:** Production-ready (FactAtom only)
**Module:** `backend/extraction/`

---

## 1. Purpose

Transform a `CanonicalDocument` (output of Slice 1.2) into structured knowledge
atoms using a **cheap-first extraction cascade**. This slice ships exactly one
atom type ‚Äî `FactAtom` ‚Äî while establishing the contracts, pipeline, scoring,
and ranking infrastructure that future atom types plug into without
modification.

## 2. Scope

### In scope
- `FactAtom` contract + `ExtractionResult` + `ExtractionContext`
- Three-stage cascade: rule-based, heuristic, LLM (interface only)
- Confidence engine with per-method bands
- Ranking layer (dedup + merge)
- Observability metrics
- Failure handling

### Explicitly out of scope (constraints)
- **No LLM calls.** Stage 3 is an interface plus a no-op default.
- **No database persistence.**
- **No entity resolution.**
- **No vector indexing.**
- **No graph generation.**
- Atom types `EntityAtom`, `RelationshipAtom`, `SignalAtom`, `EventAtom` are
  declared in the `AtomType` enum as extension points but are never emitted.

## 3. Position in the ingestion flow

```
Connector ‚Üí Normalization ‚Üí PII Screening ‚Üí Injection Screening
          ‚Üí CanonicalDocument ‚Üí [ EXTRACTION ENGINE (this slice) ] ‚Üí ExtractionResult
```

The engine treats `CanonicalDocument` as read-only input and emits an
in-memory `ExtractionResult`. Nothing is persisted; downstream slices consume
the result.

## 6. Cheap-first cascade

Stages run cheapest ‚Üí most expensive, short-circuiting as soon as a cheap stage
succeeds.

1. **RuleBasedExtractor** ‚Äî deterministic, high precision. Confidence ‚â• 0.90.
   Templates: `X upgraded to Y`, `X belongs to Y`, `X started at Y`, plus
   `downgraded_to`, `is_a`, `acquired`.
2. **HeuristicExtractor** ‚Äî runs **only when Stage 1 returns nothing**.
   Sentence segmentation, noun-phrase detection, verb-pivot relation
   inference. Confidence 0.60‚Äì0.80.
3. **LLMExtractor** ‚Äî interface only. **Never invoked in S1.3.** A future
   slice wires Gateway routing; the default `NullLLMExtractor` emits nothing.

This ordering minimizes cost: most documents are satisfied by deterministic
rules; only ambiguous text would fall through to (eventually) the LLM.

## 7. Confidence model

| Method | Band | Default |
|---|---|---|
| Rule based | 0.90 ‚Äì 1.00 | 0.92 |
| Heuristic | 0.60 ‚Äì 0.80 | 0.65 |
| Future LLM | 0.70 ‚Äì 0.95 | 0.80 |

The `ConfidenceScorer` validates extractor output against the band for its
method and clamps out-of-band values defensively, so an extractor bug cannot
emit an invalid score. The pipeline infers method from the confidence value
(‚â• 0.90 ‚áí rule-based, else heuristic) because the public `Atom` contract does
not carry a method field; the producing stage is the source of truth via its
band assignment.

## 8. Ranking

`AtomRanker` collapses atoms sharing a `dedup_key()` (tenant + document +
type + normalized subject/predicate/object). It retains the highest-confidence
instance and **unions source offsets** from all duplicates so provenance is
never lost. Output ordering is deterministic: descending confidence, then
dedup key.

## 9. Observability

| Metric | Type | Labels |
|---|---|---|
| `extraction_documents_total` | counter | tenant |
| `extraction_atoms_total` | counter | tenant |
| `extraction_failures_total` | counter | stage |
| `extraction_latency_ms` | histogram | tenant |

Metrics flow through a `MetricsSink` interface. `NullMetricsSink` is the safe
default; `InMemoryMetricsSink` backs tests; production injects a
Prometheus/OTel sink.

## 10. Extensibility

Future atom types add a `FactAtom`-style subclass of `Atom` (defining
`dedup_key()`), optionally a template set or extractor, and a confidence band.
The pipeline, scorer, and ranker operate polymorphically and require no change.

## 11. Determinism & purity

Stages 1‚Äì2 are pure and side-effect free: no network, no disk, no clock
dependence beyond `extractedAt`. This makes extraction reproducible and unit
-testable without mocks.
