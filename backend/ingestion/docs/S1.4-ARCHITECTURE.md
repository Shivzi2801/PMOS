# S1.4 — Architecture

## Scope

Wave 1, Slice 1.4 of PMOS implements deterministic **Entity Resolution**,
**Provenance Tracking**, and **Source ACL Enforcement**. It consumes the
extracted atoms produced upstream (S1.2 / S1.3) and emits deduplicated
**canonical entities**, each carrying full source lineage and gated by
tenant-scoped access control.

## Component Map

| File | Responsibility |
|------|----------------|
| `models/canonical_entity.py` | Resolved, deduplicated entity + aliases |
| `models/provenance_record.py` | Source lineage for each contributing atom |
| `models/source_acl.py` | Tenant-scoped ALLOW/DENY visibility rules + registry |
| `engine/resolver.py` | Deterministic exact / alias / normalized matching |
| `engine/merge_pipeline.py` | ACL gate → resolve → attach provenance → emit |
| `metrics.py` | Counters: processed, merged, merge_rate, acl_rejections, provenance |
| `errors.py` | `ResolutionError`, `ACLViolationError`, `ProvenanceError` |

## Entity Lifecycle

1. **Ingest** — An `ExtractedAtom` (id, name, identity_type, source_id)
   arrives paired with a `ProvenanceRecord` as a `PipelineInput`.
2. **ACL gate** — The pipeline calls `ACLRegistry.check(source_id, principal)`.
   Cross-tenant and unregistered sources fail closed (DENY). Denied atoms
   are dropped and counted as `acl_rejections`; they never reach the resolver.
3. **Resolve** — The surviving atom is matched against the in-run index by
   exact → alias → normalized strategy. A match folds the atom into the
   existing entity (a *merge*); no match mints a new entity.
4. **Provenance attach** — The atom's `ProvenanceRecord` is appended to the
   resolved entity's lineage list, keyed by `entity_id`.
5. **Emit** — `run()` returns all canonical entities, their provenance map,
   the rejected atom ids, and a metrics snapshot.

## Canonical Entity Creation

A canonical id is derived deterministically:

```
normalized = normalize_text(name)         # lowercase, strip punct, drop org suffixes
canonical_id = "ent_" + sha256(f"{identity_type}:{normalized}")[:24]
```

Because the id depends only on identity type and normalized text — never on
ordering or wall-clock time — the same logical entity always resolves to the
same id across runs, processes, and machines. `Acme Corp`, `ACME Corporation`,
and `Acme` all normalize to `acme` and therefore share one canonical id.

## Provenance Graph

Each canonical entity is the root of a small lineage graph:

```
CanonicalEntity(entity_id)
   ├── ProvenanceRecord(extraction_id=a1, source_id, document_id, connector_type, ...)
   ├── ProvenanceRecord(extraction_id=a2, ...)
   └── ProvenanceRecord(extraction_id=aN, ...)
```

The pipeline stores this as `Dict[entity_id, List[ProvenanceRecord]]`. Every
record retains `tenant_id`, `connector_type`, `connection_id`, `document_id`,
and the literal `evidence_text`, so any resolved fact can be traced back to the
exact source span that supported it. A record with no `source_id` is an orphan
and is rejected at construction time.

## ACL Enforcement

ACL checks run **before** resolution by design: denied evidence must never be
able to influence a canonical entity that another principal could later read.
`SourceACL.evaluate` applies precedence:

1. Cross-tenant → DENY (always).
2. Explicit per-principal rule (ALLOW/DENY) → authoritative.
3. Source `default_decision`.

The `ACLRegistry` fails closed: a source with no registered ACL is DENY. This
guarantees that an unknown or mis-wired source cannot leak into shared
canonical state.

## Determinism Guarantees

- Normalization is pure and order-independent.
- Canonical ids are content-addressed hashes.
- Merge confidence is the conservative minimum across contributing matches.
- Given the same atom set, the emitted entity set and ids are identical
  regardless of input ordering (verified by `test_canonical_ids_stable_across_runs`).
