# S1.8 — Answer Generation Foundation: Architecture

## 1. Purpose & scope

The Generation layer (S1.8) consumes the **PromptPackage** produced by S1.7
(Context Assembly) and produces a **GeneratedAnswer**: a grounded,
citation-aware response together with the provenance and observability data
needed by downstream slices.

S1.8 owns five concerns:

1. A **provider-agnostic LLM abstraction** (OpenAI, Azure OpenAI, Anthropic,
   local OpenAI-compatible servers, and a deterministic mock).
2. An **orchestration pipeline** with retry/transient-failure handling.
3. **Grounded-response enforcement** — answers only from supplied context.
4. **Citation binding** — mapping generated statements back to source records.
5. **Safety controls** and **observability**.

S1.8 does *not* perform retrieval, ranking, or prompt assembly — those belong
to S1.6/S1.7. It treats the inbound `PromptPackage` as an immutable contract.

## 2. Module layout

```
generation/
  contracts/        Immutable value objects + typed error taxonomy
    models.py         PromptPackage, ContextChunk, CitationRecord,
                      LLMRequest/Response, GeneratedAnswer, AnswerStatus
    errors.py         GenerationError hierarchy (transient vs permanent)
  providers/        LLM abstraction layer
    base.py           LLMProvider interface + registry
    mock.py           Deterministic offline provider (tests/local)
    openai_provider.py    OpenAI + Azure OpenAI adapters
    anthropic_provider.py Anthropic + local HTTP adapters
  pipeline/         Orchestration
    prompt_builder.py Renders PromptPackage -> LLMRequest
    retry.py          Bounded exponential back-off policy
    pipeline.py       GenerationPipeline (the orchestrator)
  grounding/
    enforcer.py       Grounding verdict + confidence scoring
  citation/
    binder.py         Inline marker parsing + CitationRecord binding
  safety/
    controls.py       Pre-flight gates + terminal status evaluation
  observability/
    metrics.py        MetricsSink interface + in-memory/null sinks
```

This mirrors the layered, contract-first structure used in S1.1–S1.7: a pure
`contracts` core, side-effect-free domain modules (grounding, citation,
safety), an injectable `observability` façade, and a thin orchestration layer.

## 3. Data flow

```
S1.7 PromptPackage
      │
      ▼
 [safety.preflight]  ── empty context / score floor ──▶ terminal answer
      │ allowed
      ▼
 [prompt_builder.build_request]  ──▶ LLMRequest (neutral)
      │
      ▼
 [pipeline._invoke_with_retry] ──▶ provider.generate() ──▶ LLMResponse
      │            ▲ retry on TransientProviderError (bounded)
      ▼
 [citation.bind_citations]  ──▶ BindingResult (bound, supporting, unknown)
      │
      ▼
 [grounding.enforce]  ──▶ GroundingVerdict (grounded?, confidence, refusal?)
      │
      ▼
 [safety.evaluate]  ──▶ AnswerStatus (OK / REFUSED / UNSUPPORTED / LOW_CONFIDENCE)
      │
      ▼
   GeneratedAnswer  ──▶ caller
```

Every transition emits metrics through the injected `MetricsSink`.

## 4. LLM abstraction layer

`LLMProvider` is the single seam between the pipeline and any vendor. Concrete
adapters translate the neutral `LLMRequest`/`LLMResponse` to and from native
SDK calls, and **normalise transport errors** into the `ProviderError`
taxonomy so the retry policy is provider-independent.

- **Lazy SDK imports.** Vendor SDKs (`openai`, `anthropic`) are imported inside
  the adapter, so the core package has no hard third-party dependency and the
  mock/local providers run with the stdlib alone.
- **Configurable selection.** `create_provider(name, **kwargs)` instantiates by
  registry name; `GenerationParams.model` selects the model per request, with
  `provider.default_model` as fallback.
- **Dependency injection for tests.** Each adapter accepts an injectable client
  (or HTTP transport for `local`) so unit tests never touch the network.

| Provider        | Registry name   | Model selector            |
|-----------------|-----------------|---------------------------|
| OpenAI          | `openai`        | model name                |
| Azure OpenAI    | `azure_openai`  | deployment name           |
| Anthropic       | `anthropic`     | model name                |
| Local (vLLM/Ollama/llama.cpp) | `local` | model name (OpenAI-compat) |
| Mock            | `mock`          | n/a (deterministic)       |

## 5. Grounded-response enforcement

Grounding is enforced both **by construction** and **by inspection**:

- *By construction:* `prompt_builder` injects a strict grounding directive and
  renders each chunk with an explicit `[chunk_id]` marker, instructing the
  model to cite only those markers and to emit a fixed refusal phrase when the
  context is insufficient.
- *By inspection:* after generation, `grounding.enforce` rejects answers that
  (a) cite markers absent from the supplied context (hallucinated citations),
  or (b) assert substantive content while citing nothing. Only chunks actually
  cited are recorded in `supporting_chunk_ids`.

## 6. Citation binding

`citation.binder` parses inline `[chunk_id]` markers, binds each to the
`CitationRecord` S1.7 supplied for that chunk, and records character offsets so
callers can render inline citation metadata. Markers referencing unknown chunks
are surfaced as `unknown_markers` for the grounding layer to act on.

## 7. Safety controls

- **Pre-flight gates** (before any provider call): empty-context →
  `EMPTY_CONTEXT`; mean retrieval score below `min_context_score` →
  `LOW_CONFIDENCE`. These avoid wasted, ungroundable LLM calls.
- **Post-generation evaluation** maps the grounding verdict + confidence onto a
  terminal `AnswerStatus`, applying `confidence_threshold`.

Confidence is a deterministic heuristic (citation coverage × mean retrieval
score, penalised by hallucinated markers). It is intentionally explainable and
testable; a future revision can substitute model-reported logprobs without
changing the contract.

## 8. Observability

The pipeline emits, through an injectable `MetricsSink`:

- `generation.requests`, `generation.failures` (tagged by `kind`),
  `generation.retries` (tagged by `attempt`) — counters
- `generation.latency_ms` (tagged by `status`) — timing
- `generation.tokens.{prompt,completion,total}` — distributions

`NullMetricsSink` is the default; `InMemoryMetricsSink` backs tests;
production binds a StatsD/Prometheus/OTel implementation.

## 9. Design principles (consistent with S1.1–S1.7)

- **Contract-first & immutable.** All boundary types are frozen dataclasses.
- **Pure domain core, injected side-effects.** Metrics and sleep are injected;
  the domain logic is deterministic and unit-testable end to end via the mock.
- **Explicit failure taxonomy** rather than string matching on exceptions.
- **No hidden global state** beyond the provider registry, which is populated
  at import time and overridable.

## 10. Extension points

- New provider: implement `LLMProvider`, call `register_provider`.
- New grounding/confidence model: replace `grounding.enforcer` internals; the
  `GroundingVerdict` contract is stable.
- Streaming: add a `generate_stream` method to `LLMProvider` and a streaming
  variant of the pipeline; the citation/grounding layers operate on the final
  assembled text and are unaffected.
