# S1.8 — Sequence Diagrams

All diagrams use Mermaid. Participants map directly to modules in
`generation/`.

## 1. Happy path — grounded answer

```mermaid
sequenceDiagram
    autonumber
    participant Caller
    participant Pipe as GenerationPipeline
    participant Safe as safety.controls
    participant PB as prompt_builder
    participant Prov as LLMProvider
    participant Bind as citation.binder
    participant Gr as grounding.enforcer
    participant Met as MetricsSink

    Caller->>Pipe: generate(PromptPackage)
    Pipe->>Met: increment(requests)
    Pipe->>Safe: preflight(package)
    Safe-->>Pipe: allowed
    Pipe->>PB: build_request(package)
    PB-->>Pipe: LLMRequest
    Pipe->>Prov: generate(LLMRequest)
    Prov-->>Pipe: LLMResponse(text, usage)
    Pipe->>Met: observe(tokens.*)
    Pipe->>Bind: bind_citations(text, package)
    Bind-->>Pipe: BindingResult(bound, supporting, unknown=∅)
    Pipe->>Gr: enforce(package, binding, text)
    Gr-->>Pipe: verdict(grounded=true, confidence)
    Pipe->>Safe: evaluate(package, verdict)
    Safe-->>Pipe: AnswerStatus.OK
    Pipe->>Met: observe(latency_ms, status=ok)
    Pipe-->>Caller: GeneratedAnswer(OK, citations, support)
```

## 2. Transient provider failure with retry

```mermaid
sequenceDiagram
    autonumber
    participant Pipe as GenerationPipeline
    participant Prov as LLMProvider
    participant Retry as RetryPolicy
    participant Met as MetricsSink

    Pipe->>Prov: generate(req)  %% attempt 1
    Prov--xPipe: TransientProviderError (429)
    Pipe->>Retry: is_retryable? -> true
    Pipe->>Met: increment(retries, attempt=1)
    Pipe->>Retry: delay_for(1)
    Pipe->>Pipe: sleep(delay)
    Pipe->>Prov: generate(req)  %% attempt 2
    Prov-->>Pipe: LLMResponse
    Note over Pipe: continue to binding/grounding
```

## 3. Retry exhausted → provider error

```mermaid
sequenceDiagram
    autonumber
    participant Pipe as GenerationPipeline
    participant Prov as LLMProvider
    participant Met as MetricsSink

    loop attempts 1..max_attempts
        Pipe->>Prov: generate(req)
        Prov--xPipe: TransientProviderError
    end
    Pipe->>Met: observe(latency_ms, status=provider_error)
    Pipe->>Met: increment(failures, kind=provider_error)
    Pipe-->>Pipe: terminal(PROVIDER_ERROR)
```

```mermaid
    Bind-->>Pipe: BindingResult(unknown=[c42])
    Pipe->>Gr: enforce(...)

```
