# S1.7 — Context Assembly — Failure Handling

This layer fails **loudly, early, and locally**. It performs no I/O, so all
failure modes are deterministic functions of the input and configuration. Every
error derives from `ContextError`, allowing a single catch at the Generation
boundary while preserving discriminability.

## 1. Error hierarchy

```
ContextError
├── InvalidRetrievalResultError   # malformed S1.6 input
├── TokenBudgetError              # invalid/unsatisfiable budget config
├── EmptyContextError             # nothing survived filtering/budgeting
├── PromptTemplateError           # unknown or invalid template
└── CitationError                 # citation could not be constructed
```

## 2. Failure modes and handling

| # | Condition | Stage | Behavior | Error |
|---|-----------|-------|----------|-------|
| 1 | `result is None` | validate | Reject immediately | `InvalidRetrievalResultError` |
| 2 | Empty/blank query | validate | Reject immediately | `InvalidRetrievalResultError` |
| 3 | `chunks is None` | validate | Reject immediately | `InvalidRetrievalResultError` |
| 4 | Chunk not a `RetrievedChunk` | validate | Reject immediately | `InvalidRetrievalResultError` |
| 5 | Empty `chunk_id` | validate | Reject immediately | `InvalidRetrievalResultError` |
| 6 | `max_context_window <= 0` | budget construction | Reject at construction | `TokenBudgetError` |
| 7 | Negative reservation | budget construction | Reject at construction | `TokenBudgetError` |
| 8 | Reservations ≥ window | budget construction | Reject at construction | `TokenBudgetError` |
| 9 | All chunks duplicates/ACL-blocked/oversized | budget | No selection | `EmptyContextError` (unless `allow_empty_context`) |
| 10 | Unknown template name | prompt build | Reject | `PromptTemplateError` |
| 11 | Single chunk exceeds whole budget | budget | Chunk dropped, scan continues | none (recorded in metrics + `dropped_chunk_ids`) |

## 3. Graceful degradation vs. hard failure

- **Hard failure (raise).** Contract violations (modes 1–8, 10) and the
  no-context case (mode 9) raise. These represent states where producing a
  prompt would be incorrect or impossible.
- **Graceful degradation (no raise).** Oversized individual chunks (mode 11)
  and chunks removed by dedup/ACL are dropped silently from the output but are
  **always** accounted for: their ids appear in
  `ContextPackage.dropped_chunk_ids`, and counts are reflected in
  `AssemblyMetrics` (`duplicate_chunks_removed`, `acl_filtered_chunks`,
  `budget_dropped_chunks`).

## 4. The empty-context switch

`ContextAssembler.allow_empty_context` controls mode 9:

- `False` (default) — raise `EmptyContextError`. Recommended for production:
  generation without grounding context risks hallucination.
- `True` — return a `ContextPackage` with `is_empty == True`. Use only when the
  downstream template is explicitly designed to answer without retrieved
  context (e.g. a fallback "I could not find relevant sources" template).

## 5. Auditability guarantees

For any successful `assemble`/`build_prompt`, the following invariant holds:

```
len(input.chunks)
  == len(package.chunks)
   + len(unique-loss-from-dedup)
   + acl_dropped
   + budget_dropped
```

Equivalently, every input chunk id is either present (as a `ContextChunk`) or
listed in `dropped_chunk_ids`. No chunk vanishes silently. This makes the layer
debuggable purely from its outputs plus an `AssemblyMetrics` snapshot.

## 6. Recommended caller pattern

```python
from backend.context import (
    ContextAssembler, TokenBudget, ACLFilter, AssemblyMetrics,
    ContextError, EmptyContextError,
)

budget = TokenBudget(max_context_window=8192, reserved_response=1024)
assembler = ContextAssembler(budget=budget)
metrics = AssemblyMetrics()

try:
    package = assembler.build_prompt(
        retrieval_result,
        acl=ACLFilter.from_iterable(principal_tags),
        metrics=metrics,
    )
except EmptyContextError:
    # No grounding sources — route to fallback or surface "no answer".
    handle_no_context()
except ContextError as exc:
    # Contract/config error — log and fail the request.
    log.error("context assembly failed: %s", exc)
    raise
finally:
    telemetry.emit(metrics.snapshot())
```

## 7. Concurrency notes

- `TokenBudget`, all filters, the ranker, and every package type are immutable
  (`frozen=True`) and therefore safe to share across threads.
- `ContextAssembler` holds only immutable collaborators plus a `PromptBuilder`.
  A single `PromptBuilder` is safe to share once templates are registered;
  register all templates before concurrent use.
- `AssemblyMetrics` is **mutable and per-request**. Allocate a fresh instance
  per `assemble`/`build_prompt` call (the default when omitted) — do not share
  one instance across concurrent requests.
