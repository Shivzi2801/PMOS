# S1.7 — Context Assembly & Prompt Construction — Architecture

Feature ID: **F-15**
Wave 1 / Slice 1.7. Sits **after** Retrieval (S1.6) and **before** Generation (Wave 2).

## 1. Purpose

The Context Assembly layer transforms `RetrievalResult` objects (S1.6) into a
single, model-ready `PromptPackage` (Wave 2 input). It is a **pure domain
layer**: no LLM calls, no framework dependencies, strong typing via
`dataclasses`.

## 2. Position in the pipeline

```
[S1.6 Retrieval] --RetrievalResult--> [S1.7 Context Assembly] --PromptPackage--> [Wave 2 Generation]
```

The layer never reaches outward. It consumes the S1.6 contract and emits a
transport-neutral package that a Generation adapter renders into a concrete
provider request.

## 3. Component map

| Component | File | Responsibility |
|-----------|------|----------------|
| `RetrievedChunk`, `RetrievalResult` | `context_package.py` | S1.6 contract (consumed, field-compatible shim) |
| `ContextChunk`, `ContextPackage` | `context_package.py` | Selected, ordered, marker-annotated context |
| `PromptPackage`, `PromptMessage` | `prompt_package.py` | Terminal artifact for Generation |
| `CitationRecord` | `citation_record.py` | Source attribution per selected chunk |
| `TokenBudget` | `token_budget.py` | Context-window partitioning + token counting |
| `DeduplicationFilter`, `ACLFilter` | `context_filters.py` | Dedup + access-control filtering |
| `ContextRanker` | `context_ranker.py` | Relevance ordering (deterministic) |
| `PromptBuilder`, templates | `prompt_builder.py` | Renders `ContextPackage` -> `PromptPackage` |
| `ContextAssembler` | `context_assembler.py` | Orchestrates the full pipeline |
| `AssemblyMetrics` | `metrics.py` | Observability counters/timers |
| Errors | `errors.py` | Domain error hierarchy |

## 4. Processing stages (in order)

1. **Validate** the incoming `RetrievalResult` (non-empty query, well-typed
   chunks, non-empty chunk ids).
2. **Deduplicate** by chunk id, then by normalized content hash. First
   occurrence wins.
3. **ACL filter** — drop chunks whose access-control tags do not intersect the
   principal's granted tags. Untagged chunks are public.
4. **Rank** by relevance score, descending, with deterministic id tie-break.
5. **Enforce token budget** — greedy fill against `available_for_context`;
   chunks that do not fit are dropped, smaller later chunks may still be kept.
6. **Construct context window** — assign sequential markers `[1]`, `[2]`, ...
   and compute per-chunk token cost.
7. **Generate citations** — one `CitationRecord` per selected chunk, carrying
   document id, source URI, title, and score.
8. **Build prompt** — a registered template renders messages; token estimate
   and response reservation are attached -> `PromptPackage`.

## 5. Token budget model

```
max_context_window
  = reserved_system + reserved_query + reserved_citations
  + reserved_response + available_for_context
```

`available_for_context` is the only region selection draws from. Token counting
is pluggable via a `TokenCounter` callable; a dependency-free heuristic
(~4 chars/token, blended with word count) is the default. A real BPE tokenizer
can be injected without touching any other component.

## 6. Design principles

- **Purity & determinism.** Same input + same budget -> byte-identical output.
  Ranking ties break on chunk id; markers are positional.
- **No outward calls.** The layer cannot perform I/O or call models; this keeps
  it unit-testable and side-effect free.
- **Strong typing.** Every boundary object is a frozen `dataclass`. Mutable
  state is confined to `AssemblyMetrics` and the `PromptBuilder` registry.
- **Accountability.** Every dropped chunk id is retained in
  `ContextPackage.dropped_chunk_ids`, and counts flow into `AssemblyMetrics`.

## 7. Extensibility

- **Prompt templates.** Implement the `PromptTemplate` protocol (a `name` plus
  `build(context, budget) -> list[PromptMessage]`) and register it with
  `PromptBuilder.register_template`. The default is `qa_grounded`. The
  assembler and all other stages are untouched.
- **Ranking strategy.** Inject a custom `ContextRanker` (e.g. recency-weighted)
  via the assembler constructor.
- **Tokenizer.** Inject a `token_counter` into `TokenBudget`.
- **Filters.** `DeduplicationFilter` and `ACLFilter` are independent objects;
  additional filter stages can be composed in the assembler.

## 8. Contract compatibility with S1.6

`RetrievedChunk` / `RetrievalResult` mirror the upstream Retrieval output and
are referenced only by attribute name. If the concrete S1.6 module is
importable in the deployed environment, those types may be substituted directly
because assembly code performs structural (duck-typed) access on
`chunk_id`, `document_id`, `content`, `score`, `acl_tags`, `source_uri`,
`title`, and `metadata`.
