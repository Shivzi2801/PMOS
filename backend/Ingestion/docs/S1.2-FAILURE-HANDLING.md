The pipeline is **fail-safe**: when in doubt, a record is withheld from the safe
stream rather than published. No failure mode results in unsafe content being
silently published.

## 1. Failure modes and responses

| # | Failure | Stage | Response |
|---|---|---|---|
| 1 | Missing required field (e.g. ticket `id`) | Normalization | `NormalizationError` â†’ `PipelineResult{FAILED}` with error message; nothing published |
| 2 | Unknown `connector_type` | Normalization | `NormalizationError`; surfaced as `FAILED`; operator must register a normalizer |
| 3 | Malformed / non-dict payload | Normalization | `NormalizationError`; `FAILED` |
| 4 | PII detector false negative | PII | Defense in depth: injection screening + CRITICAL-PII rule still apply; severity ladder errs toward caution |
| 5 | PII detector false positive | PII | Non-destructive in ANNOTATE mode; in REDACT mode only the matched span is masked, body remains usable |
| 6 | Injection borderline score | Injection | `SUSPECT` status â†’ quarantined by default (`quarantine_on_suspect=True`) |
| 7 | Hidden / zero-width payload | Normalization + Injection | Stripped from canonical body during normalization **and** flagged by the screener on raw text |
| 8 | Quarantine store write error | Quarantine | Exception propagates to caller; record is **not** marked published â€” caller retries; use durable `FileQuarantineStore` in production |
| 9 | Empty body / title | All | Empty text yields `SAFE` / no findings; record published as-is |

## 2. Design principles

- **Conservative defaults.** `SUSPECT` is treated like `QUARANTINED` unless an
  operator explicitly opts out. CRITICAL PII (live secrets) always quarantines,
  even with no injection signal.
- **Provenance is never lost.** Every quarantined record stores full provenance
  plus the verbatim `original_payload`, enabling audit and safe re-ingestion
  after a false positive is cleared.
- **No partial publish.** A record is either fully published (after redaction
  and annotation) or fully quarantined. There is no half state.
- **Idempotency aid.** `CanonicalDocument.content_hash` lets downstream
  consumers dedupe re-processed records.

## 3. Re-ingestion after a false positive

1. Operator inspects the `QuarantineRecord` (reason, findings, payload).
2. If benign, the `original_payload` is re-submitted to the pipeline, optionally
   with adjusted thresholds or `quarantine_on_suspect=False`.
3. The new run produces a fresh `PipelineResult`; the original quarantine record
   is retained for audit.

## 4. Observability hooks

- `PipelineResult.to_dict()` is fully serializable for structured logging.
- `InjectionFinding.score` and per-`InjectionMatch` weights make threshold
  tuning measurable.
- `QuarantineService.count()` / `list()` expose backlog size for alerting.

## 5. Security notes

- Detector `preview` fields are **masked** (e.g. `AKIA****â€¦`, `***-**-6789`) so
  logging findings never leaks the underlying secret.
- Zero-width characters are removed before the canonical body is persisted,
  preventing stored hidden-prompt payloads.
- The PII `REDACT` mode replaces secrets in the published body with typed
  placeholders (`[REDACTED:SSN]`).
