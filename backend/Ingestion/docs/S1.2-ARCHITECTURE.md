# S1.2 — Ingestion Safety Pipeline: Architecture (F-09)

Wave 1 · Slice 1.2

## 1. Purpose

Slice 1.2 turns raw connector records (produced by the Slice 1.1 Connector SDK
and Zendesk connector) into **safe canonical PMOS documents**. Any record that
carries live secrets or prompt-injection payloads is automatically held back in
a quarantine store instead of entering the safe stream.

## 2. Pipeline

```
Raw Connector Record
   │
   ▼
[1] Normalization ─────► CanonicalDocument (clean body, provenance, metadata)
   │
   ▼
[2] PII Detection ─────► findings (type, confidence, severity) + optional redaction
   │
   ▼
[3] Injection Screening ► InjectionFinding { status: SAFE | SUSPECT | QUARANTINED }
   │
   ▼
[4] Decision
   ├── safe  ─────────► Safe Canonical Document (PUBLISHED, PII redacted, annotated)
   └── unsafe ────────► Quarantine (QUARANTINED, provenance + raw payload preserved)
```

## 3. Modules

| Module | Path | Responsibility |
|---|---|---|
| Contracts | `backend/ingestion/contracts/` | Shared serializable shapes: `CanonicalDocument`, `PIIFinding`, `InjectionFinding`, `QuarantineRecord`, enums |
| Normalization | `backend/ingestion/normalization/` | Connector payload → `CanonicalDocument`; registry + per-connector normalizers |
| PII | `backend/ingestion/pii/` | Detectors + engine; redact / annotate modes |
| Injection | `backend/ingestion/injection/` | Pattern catalogue + scoring screener |
| Quarantine | `backend/ingestion/quarantine/` | Store backends + service |
| Pipeline | `backend/ingestion/pipeline/` | Orchestrates the four stages + decision policy |

## 4. Contracts (canonical shapes)

- **CanonicalDocument** — `title`, `body`, `provenance`, `source_metadata`,
  `document_id`, `content_hash`, `annotations`. Provenance and connector
  identifiers are mandatory and never dropped.
- **Provenance** — `connector_id`, `connector_type`, `source_id`, `source_type`,
  `source_url`, source timestamps, `ingested_at`, `sync_cursor` (from Slice 1.1).
- **PIIFinding** — `pii_type`, `severity`, `confidence (0–1)`, span, masked
  `preview`, `detector`.
- **InjectionFinding** — `status`, `score`, `matches[]` with `category`.
- **QuarantineRecord** — `provenance`, `reason`, `original_payload` (verbatim),
  `injection_finding`, `pii_findings`, `canonical_snapshot`.

## 5. Detection design

### PII
Six detectors: Email, Phone, SSN, Credit Card, API Key, Access Token. Detectors
pair regex with validation (Luhn for cards, SSA range rules for SSNs, vendor key
prefixes for secrets) to raise confidence and cut false positives. Overlapping
spans are de-duplicated keeping the highest-confidence finding. Severity ladder:
`LOW (email) < MEDIUM (phone) < HIGH (SSN, card) < CRITICAL (API key, token)`.

### Injection
Five categories — instruction override, system-prompt extraction, data
exfiltration, tool misuse, hidden prompt (incl. zero-width characters). Each
matched pattern contributes a weight; the total is a risk **score** with a
diversity bonus for distinct categories and diminishing returns on repeats.
Thresholds map score → status: `SAFE (<0.45)`, `SUSPECT (<0.85)`,
`QUARANTINED (≥0.85)`.

## 6. Decision policy

A record is **quarantined** when any of the following hold:
1. Injection status is `QUARANTINED`.
2. Injection status is `SUSPECT` (conservative default; configurable).
3. Any **CRITICAL** PII (API key / access token) is present.

Otherwise it is **published**: non-critical PII is redacted in the body, and PII
+ injection summaries are attached to `annotations`.

## 7. Configuration knobs

- `PIIEngine(min_confidence=…)`, custom detector list.
- `InjectionScreener(suspect_threshold, quarantine_threshold, hidden_char_weight)`.
- `IngestionPipeline(pii_mode=REDACT|ANNOTATE, quarantine_on_suspect=bool)`.
- Quarantine backend: `InMemoryQuarantineStore` or durable `FileQuarantineStore`.

## 8. Extensibility

New connectors implement `Normalizer` and register with the
`NormalizerRegistry`; the rest of the pipeline is connector-agnostic because it
operates only on `CanonicalDocument`.
